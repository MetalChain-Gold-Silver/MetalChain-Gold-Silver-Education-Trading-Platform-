# Security Policy

## Supported Versions

MetalChain is pre-release. Security fixes will be handled on the latest active branch.

## Reporting a Vulnerability

Please do not open public issues for sensitive vulnerabilities.

Until a private security channel exists, configure a maintainer email or security inbox before making the repository public.

Suggested process:

1. Share a private report with reproduction details
2. Allow maintainers time to verify the issue
3. Coordinate a fix and disclosure timeline

## Scope Notes

Because MetalChain may eventually touch financial workflows, security-sensitive areas include:

- authentication
- wallet integrations
- transaction signing
- portfolio balances
- market data integrity
- offline transaction sync
